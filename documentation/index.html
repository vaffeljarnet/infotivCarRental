This project was created at an internship at Infotiv AB, Gothenburg, by Joakim Gustavsson and Johan Larsson, with the help of Maheel Dabarera. The project consists of a fake car rental booking homepage, to be used as a system under test for educational purposes.

This documentation will explain options on how to set the homepage up locally and externally, explanations of functionality and standards used over the different pages. Use the category selector to the left to jump to a specific chapter.

How To Run#back to top

If you wish only to view the homepage, it's available here: http://35.234.77.201/

If you want to further develop upon our groundwork, you'll find below our methods of setting up and running the homepage locally and on Google Cloud Platform. Off course there are alternatives to this, however, we will not go into these in this documentation, but on a basic ground level the requirements to run the homepage are:

  1. Source code: GitHub
  2. Server to host the files
  3. MySQL database on that server

Local Server - #back to top

When starting to develop this project we ran it locally using MAMP. This has proven to be useful to be able to test our changes before deploying them on a server. Below are instructions on how we set up our MAMP configuration.

  1. Download and install MAMP with default values: MAMP Download
  2. Download the source code from GitHub, extract content and put the folder infotivCarRental in C:\MAMP\htdocs
  3. Start MAMP and wait for Apache and MySQL server to start
  4. Open your browser and go to: http://localhost/phpMyAdmin
  5. Click the tab User Accounts and select Edit Privileges on the row corresponding to User name: root Hostname: localhost
  6. Press Change password, select a new password and take note of it.
  7. In a text editor, open the file C:\MAMP\bin\phpMyAdmin\config.inc.php
  8. Go to line 61, change the password from root to the one you entered in phpMyAdmin, save the file and exit
  9. In a text editor, open the file C:\MAMP\htdocs\infotivCarRental\html\includes\dbh.inc.php.template
  10. Go to line 7, change the password to the one you entered in phpMyAdmin
  11. Rename the file as dbh.inc.php and save it in the same directory
  12. Go back to phpMyAdmin in your browser and press New in the left section of the page
  13. Input fleet_information as name, press Create and a new database should be added in the left section of the page
  14. Make sure the database is selected and press the tab Import in the top of the page
  15. Press Select file and browse for C:\MAMP\htdocs\infotivCarRental\fleet_information.sql
  16. When the file is selected, scroll down, press Go and the available tables should be added to fleet_information

If everything is in order, you should now be able to use the homepage by going to http://localhost/infotivCarRental/html/gui/index.php

External Server - #back to top

When setting up our homepage on Google Cloud Platform the following step by step tutorial from Google was used: Setting Up LAMP on Compute Engine

It will instruct you on how to setup an Apache server with MySQL database on a compute engine, and for our purposes all the steps in the guide were appropriate.

Depending on your operating system, transfering the files to the compute engine might cause some issues. An alternative to the method used in the guide is to upload the files to a bucket on Google Cloud Platform and then transfering them to the compute engine. A tutorial for this can be found here: Transfering Files to Instances

You will need the source code as well, which you can download here: GitHub

In the files you need to make one edit, the password for the database connection. When setting up the database according to the guide above you will be asked to set a root password for phpMyAdmin, take note of this password, and among the source code files:

  1. In a text editor, open the file C:\MAMP\htdocs\infotivCarRental\html\includes\dbh.inc.php.template
  2. Go to line 7, change the password to the one you entered in phpMyAdmin
  3. Rename the file as dbh.inc.php and save it in the same directory

Hopefully this will point you in the right direction of deploying the homepage on Google Cloud Platform

Page Functionality#back to top

Header - #back to top

The header is available on every page. The logo and title are clickable and will take you to the start page. The About text will take you to the about page. To the right is the user information field. If not signed in, a input field for email and password is visible in addition to the buttons Login, that will attempt to login the user, and Create user, that will take the user to the Create User page.

If one of the input fields are left empty, login will not be possible and an alert for corresponding field will appear. If the user inserts the wrong information a error message appears to the left of the buttons.

If user signs in successfully, the user information field will change to view a welcome phrase with the users firstname and the buttons will be changed to Logout, that wiill logout the user from the session, and My Page, that will take the user to the page where they can view their bookings.

Date Selection - #back to top

On the date selection page, user is required to input start and end dates for the trip. The default values will always be put as todays date. Restrictions for the start date are:

  1. Earliest selectable date is today
  2. Latest selectable date is one month ahead from today

Restrictions for end date are:

  1. Earliest selectable date is the date selected as start date
  2. Latest selectable date is one month ahead from the selected start date

The reset button will refresh the page and set dates to today. If valid dates are inputted, the Continue button will take the user to the car selection page. If invalid dates are inputted, an alert will pop up for the date field that has issues and not allow user to continue.

Car Selection - #back to top

On the car selection page, the user is required to pick a car model. The listed car models are based on whether a car of that model is available during the date range selected by the user. This is calculated with the table named bookings in the database, where every booking is registered with a date range paired with a license number of the selected car.

The cars in the list are given an image based on what model is listed, if the car model does not have a specific image available, a stock image will appear in that place.

The Select button will select the car model on the corresponding row and take the user to the confirm booking page. The Back to date selection button will take the user back to the date selection page.

If a user tries to get to this page without setting any dates, an error message will appear and no cars will be listed.

Confirm Booking - #back to top

On the booking confirmation page, the selections made by the user will be displayed. A Cancel button, that will take the user back to the car selection page, and a Confirm button that will attempt to finalize the booking.

If the user is not signed in, a alert popup will appear if the confirm button is pressed, asking the user to either sign in or create a user.

Successful Booking - #back to top

On the successful booking page, a message will appear with details of the registered booking if successful. A Home button, that will take the user to the date selection page, and a My Page button that will take the user to the page where they can view their bookings.

On a successful booking, an entry is added to the database table bookings with license number of the selected car, the selected start/end dates and the users internal id.

If the user tries to get to this page from any other place than the booking confirmation page, a error message will appear. The same will happen if user updates the page.

Create User - #back to top

On the create user page, the user may create a user account to be able to book a car. Required fields are, first and last name, phone number, email and password. A Cancel button, that will take the user to the date selection page, and a Create button that will attempt to create the user account with the inputted information.

If one of the fields are left empty, the inputted email or password does not match, the email is not in a correct format or the password is less than six characters, a alert popup will appear in the corresponding field when the Create button is pressed.

My Page - #back to top

On the my page, the user will see their upcoming bookings along with information about the booking. The information provided is an order id, brand/model of car, No passengers, licenseplate and start/end date for the order.

Next to the orders there's a button for unbooking the car which will give the user a warning/confirmation box, asking if they really do want to cancel the order.

Below the user have buttons for showing/hiding their order history which will provide the same info as above but without the option to cancel.

If a person attempts to reach mypage without being logged in they will be redirected back to the front page.

Admin Page - #back to top

On the admin page, the user will see all current bookings along with information about them. The information provided is an order id, brand/model of car, No passengers, licenseplate and start/end date for the order.

Next to the orders there is a "Complete order" button for confirming that the car has been returned and payed for. it will prompt a confirm message whether the car really has been returned. Upon agreeing the order will be moved from bookings in the database to orderhistory and will no longer be shown in the current orders table.

Above the user will have a button for adding cars to the fleet and a search field for orders. Entering an order id into the searchfield will give the user information about the order and the person that is tied to the order. upon entering an order id from ongoing orders or from completed orders the user will get information about: the users first/last name, phone number, Email adress, start/end date for the order and also the cars licenseplate.

If a person attempts to reach the admin page without being logged in or logged into an account without admin rights they will be redirected back to the front page.
